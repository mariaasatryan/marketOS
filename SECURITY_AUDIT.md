**Проблемы:**

- ❌ `SUPABASE_SETUP.md` - пароль БД в открытом виде: `maryJan12!marketOS`

**Решение:**

### ❌ 4. API: Rate Limiting и CORS

**Проблемы:**
- ❌ Нет rate limiting на серверных эндпоинтах
- ❌ CORS не настроен явно
- ⚠️ Нет валидации входных данных на некоторых эндпоинтах

**Решение:**
- Добавить `@fastify/rate-limit` для rate limiting
- Настроить CORS через `@fastify/cors` только для фронтенда
- Добавить валидацию через `@fastify/type-provider-typebox` или `zod`

---

### ⚠️ 5. Права доступа: RBAC

**Статус:**
- ✅ Supabase RLS (Row Level Security) настроен
- ❌ Серверные эндпоинты используют заглушку: `x-user-id` header (строка 16 в `analytics.ts`)
- ⚠️ Нет проверки JWT токенов на сервере

**Решение:**
- Реализовать проверку JWT токенов Supabase на сервере
- Добавить middleware для проверки ролей (RBAC)
- Убрать заглушку `x-user-id`, использовать реальную аутентификацию

---

### ❌ 6. Инфраструктура: WAF

**Статус:**
- ❌ Нет WAF (Web Application Firewall)

**Решение:**
- Настроить Cloudflare перед приложением (бесплатный план включает базовый WAF)
- Или использовать AWS WAF / Azure WAF

## 📝 Дополнительные Рекомендации

1. **Логирование:**
   - Не логировать секреты
   - Логировать попытки несанкционированного доступа

2. **HTTPS:**
   - ✅ GitHub Pages использует HTTPS автоматически
   - ⚠️ Убедиться, что серверный API тоже на HTTPS

3. **Зависимости:**
   - Регулярно обновлять зависимости (`npm audit`)
   - Использовать `npm audit fix` для исправления уязвимостей

4. **Content Security Policy (CSP):**

5. **SQL Injection:**
   - ✅ Prisma защищает от SQL injection
   - ⚠️ Проверить, что нет raw SQL запросов
